### 1.2 系统概述

本系统是一个基于 Qt 6 的重症监护室（NICU）设备模拟系统，用于模拟多种医疗监护设备的工作流程和数据采集。系统包含四个主要医疗设备模块（心电图仪、血压仪、血液透析仪、呼吸机）和一个配套的数据传输客户端。

系统采用串口通信方式，数据传输客户端模拟硬件设备发送数据，NICU 主程序接收并处理数据，实现实时监测和参数控制功能。

## 3. 需求

### 3.1 要求的状态和方式

#### 3.1.1 运行状态

- **正常运行状态**：系统启动后，各设备模块处于待机状态，等待用户操作
- **数据采集状态**：用户启动设备后，系统开始接收串口数据并实时显示
- **参数设定状态**：用户解锁参数后，可以修改设定参数
- **报警状态**：系统检测到异常时，进入报警状态，报警灯闪烁

#### 3.1.2 控制方式

- **手动控制**：用户通过按钮、下拉框等控件进行操作
- **自动控制**：系统根据设定参数自动执行流程（如血透仪的自检流程）
- **远程控制**：通过串口接收命令进行控制

| HD-002 | 支持管道清洗流程模拟 |

| TRANS-001 | 支持五个功能通道的串口数据传输 |

| TRANS-003 | 支持血压数据模拟，1000ms间隔发送 |

| TRANS-007 | 支持串口共享，呼吸、紧急和血透之间可共享串口 |

### 3.3 CSCI 外部接口需求

#### 3.3.1 串口接口

| 设备 | 串口 | 波特率 | 数据格式 |
|------|------|--------|----------|
| 心电图 | COM1 | 115200 | 12个整数，逗号分隔 |
| 血压仪 | COM3 | 115200 | 3字节二进制 |
| 呼吸机 | COM4 | 115200 | VENT_SET/VENT_STOP/VENT_DATA |
| 血透仪 | COM2 | 115200 | HDD命令/进度 |

#### 3.3.2 呼吸机串口协议

**NICU → Transmission**:
- `VENT_SET:<mode>:<tidalVolume>:<respRate>:<inspTime>:<riseTime>:<inspPressure>:<expPressure>:<backupRate>`
- `VENT_STOP`

**Transmission → NICU**:
- `VENT_DATA:<mode>:<tidalVolume>:<leak>:<minuteVol>:<respRate>:<inspTime>:<riseTime>:<inspPressure>:<expPressure>:<backupRate>:<ieRatio>`
- `e`（紧急信号）

#### 3.3.3 血透仪串口协议

**NICU → Transmission**:
- `HDD:<command>`

**Transmission → NICU**:
- `HDD:<command>:<progress>`

### 3.6 适应性需求

- 系统应支持不同分辨率的显示器
- 系统应支持不同版本的 Windows 操作系统
- 系统应具备良好的可扩展性，支持新增设备模块

### 3.7 安全性需求

- 串口通信应具备错误处理机制
- 参数设定应具备锁定机制，防止误操作
- 紧急停止功能应优先响应

### 3.8 保密性需求

- 患者数据应妥善管理
- 系统应避免泄露敏感信息

### 3.9 CSCI 环境需求

| 环境 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 |
| Qt 版本 | 6.11.1 |
| 编译器 | MinGW 8.1.0 |
| 内存 | 至少 4GB |
| 存储 | 至少 1GB 可用空间 |

### 3.10 计算机资源需求

- **CPU**: Intel Core i5 或更高
- **内存**: 4GB 或更高
- **显示**: 1920×1080 或更高分辨率
- **串口**: 虚拟串口或物理串口

|----------|------|
| 可靠性 | 系统应稳定运行，无崩溃 |
